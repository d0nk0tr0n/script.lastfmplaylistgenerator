1.4.2-beta
- Added: autocorrect=1 flag on Last.fm API calls - handles artist name
    mismatches like Beyonce → Beyoncé, Blue Oyster Cult → Blue Öyster Cult
- Added: MusicBrainz artist and track IDs passed to Last.fm API calls when
    available, bypassing name matching issues entirely
- Fixed: artist name normalized before Last.fm queries so artists stored
    with diacritics (e.g. Līve) query correctly
- Fixed: urllib.parse.quote used instead of quote_plus for artist/track
    names, preserving literal + in names like Florence + The Machine
- Fixed: getMusicBrainzArtistID() returns a list - take first element
- Fixed: genre returned as list from JSON-RPC - take first element to match
    string expected by setInfo
- Fixed: preferdifferentartist artist dedup now uses library artist name
    consistently (was comparing Last.fm name against library name)
- Fixed: addauto import guard uses startswith check to handle tagged and
    commented import variants
- Fixed: Last.fm API calls now use HTTPS instead of HTTP
- Fixed: Last.fm API calls now have a 10-second timeout and catch network
    errors gracefully instead of crashing the addon thread
- Changed: top tracks of similar artist sorted by listeners after
    playcount filter

1.4.1
- Fixed: artist existence cached per search run - artists not in the library
    are skipped after one lookup, eliminating up to four track queries per
    repeated missing artist (major speed improvement for large similar-track lists)
- Fixed: smart-quote fallback search (tier 4) skipped when track and artist
    names contain no ', ", or - characters

1.4.0
- Fixed: html.unescape() applied consistently to all artist and track names
    from Last.fm to handle XML entities like &amp; and &quot;
- Fixed: unicode diacritic stripping and smart quote normalization applied
    to both track titles and artist names before library search
- Fixed: featuring suffixes (feat., ft., featuring) stripped from track
    titles before querying Last.fm so seeding tracks with collabs return results
- Fixed: live and remaster suffixes stripped from track titles before search
- Fixed: four-tier library search fallback - exact normalized, contains
    normalized, contains original, contains smart-quote reversed
- Fixed: artist existence cached per search run - artists not in the library
    are skipped without querying all their tracks (major speed improvement)
- Fixed: similar artist API calls capped at 5 to prevent runaway requests
    when seeding from tracks with many similar artists not in the library
- Fixed: JSON-RPC queries now built with simplejson.dumps() instead of
    string interpolation, preventing injection from special characters in names
- Fixed: addedTracks type consistency - was mixing str and bytes, causing
    repeat track detection to silently fail
- Fixed: monitor loop rewritten using xbmc.Monitor correctly
- Fixed: xbmcvfs.translatePath used throughout instead of deprecated
    xbmc.translatePath
- Fixed: all file handles use with blocks
- Changed: minimalplaycount range updated to 1k/10k/50k/100k/250k/500k/1M,
    default 100,000
- Changed: log prefix standardized to [ADDON LFM]
- Changed: addon version logged at startup
- Changed: .gitattributes cleaned up for Kodi addon packaging (export-ignore)
- Added: check_normalization.sh utility for auditing Kodi music database
    for unicode/diacritic mismatches, supports SQLite and MySQL
- Added: addon shown under Music and Programs in Kodi
- Removed: dead code (fetch_searchTrack, unicode_normalize_string, dbtype
    detection, commented-out Python 2 remnants)
